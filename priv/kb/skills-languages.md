# Languages

Framed the way it should be: where each language has actually been used, and how deep the
water goes. See [Support & Technical Skills](/docs/skills-support) for the professional core
these support.

## JavaScript

The most-used language day to day, from debugging web product behavior in the browser to
building full-stack side projects. Comfortable across the stack: DOM and browser APIs,
async work, Node, and the MERN stack (see the Plot Twist project). Solid, practical
working proficiency.

## SQL

Used regularly for support investigations and application data work: querying to confirm
state, trace records, and understand what the data actually says versus what a customer
reports. Comfortable with joins, filtering, and reasoning about schema.

## Python

Reached for scripting, automation, and diagnostics: small tools that take repetitive
support work off the plate. Working knowledge of the fundamentals rather than a claim to
deep systems-Python.

## HTML / CSS

Years of supporting HTML/CSS-based publishing and output workflows, plus hand-writing the
front end of this very site. The whole design system is hand-authored CSS, no framework.
Strong practical command.

## Elixir

Currently leveling up in Elixir: this entire platform (Phoenix, LiveView, Ecto) is where
that's happening. Comfortable reading and extending the codebase and shipping real
features; still building production-scale intuition rather than claiming deep prior
mastery. The [colophon](/docs/colophon) has the honest version of this story, and it reads
as growth, not gap: the point was to ship something real while learning it.
